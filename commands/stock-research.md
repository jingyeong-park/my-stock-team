# 종목 협업 리서치 파이프라인

대상 종목: **$ARGUMENTS**

아래 흐름을 그대로 따라 실행합니다. 종목이 여러 개면 각 종목에 대해 독립적으로 반복합니다.
오늘 날짜는 항상 **현재 날짜**를 사용합니다(하드코딩 금지).

## 0. 사전 확인

- 종목명이 비어 있으면 사용자에게 분석할 종목명을 묻습니다.
- 각 종목의 **종목코드(6자리)**를 확정합니다. 모호하면 사용자에게 확인합니다.
- 재무 데이터(DART)를 쓰는 단계는 환경변수 `DART_KEY`가 필요합니다. 없으면 그 사실을 알리고, 재무 항목은 "확인 불가"로 두고 나머지는 진행합니다.

## 1. 분석 3종 병렬 실행

서로 의존성이 없으므로 **한 번에(같은 메시지에서) 병렬 호출**합니다.

- `fundamental-analyst` — DART 재무·공시 (3개년 + 직전 분기)
- `market-technical-analyst` — 주가·거래량·이동평균 추세
- `news-sentiment-analyst` — 최근 뉴스·이슈·시장 심리

각 서브에이전트 프롬프트에 **반드시 포함**할 것:
- 오늘 날짜
- 종목명 + 종목코드 (모호성 제거)
- 이미 확보한 컨텍스트가 있으면 함께 전달(중복 조회 방지)
- 산출물 규칙 리마인드(아래 가드레일)

## 2. 리스크 종합

3종 분석이 끝나면 `risk-manager`를 호출합니다.

- 서브에이전트끼리는 컨텍스트를 공유하지 않으므로, **3종 분석의 핵심 결과를 요약해 프롬프트에 직접 포함**합니다.
- 일부 분석이 비었으면 누락 사실을 명시해 전달합니다.
- pykrx로 시가총액·거래대금을 보완하도록 요청합니다(코스닥은 휴장일 보정).

## 3. 리서치 문서 작성 (reports/{종목명}.md)

위 4종 결과를 통합해 `reports/{종목명}.md`를 작성합니다. 구성:
표지(개요) → 재무 → 가격/추세 → 뉴스·심리 → 리스크 → 종합. 모든 수치에 출처·기준일을 유지합니다.

## 4. 품질 검증

`report-validation-analyst`로 `reports/{종목명}.md`를 점검합니다(정확성·일관성·완결성·근거/형식).

- 판정이 **보류(Hold)**면 지적된 치명적·중요 문제를 md에서 고친 뒤 3~4단계를 재실행합니다.
- **통과(Pass)**면 다음 단계로 진행합니다.

## 5. 리포트 내보내기 (PPTX / PDF)

`report-pptx` 스킬을 사용해 `reports/{종목명}.md`를 디자인된 리포트로 내보냅니다.
- 기본은 PPTX, 사용자가 PDF를 원하면 PDF도 생성합니다.
- 스킬이 금지 표현(투자 행동 단정)을 검출하면 해당 문구를 가드레일에 맞게 고친 뒤 재실행합니다.

## 가드레일 (모든 산출물에 항상 적용)

1. **수치마다 출처.** 모든 수치 옆에 `(출처: 데이터명, 연도/날짜)`. 출처를 붙일 수 없는 수치는 쓰지 않습니다.
2. **모르면 지어내지 않기.** 못 구한 데이터는 "확인 불가", 출처 불명 뉴스·소문은 "미확인"으로 명시합니다.
3. **투자 행동 단정 금지.** 매수/매도/보유·목표가·비중 확대/축소를 단정하지 않습니다. 판단 근거 정리(의사결정 지원)까지만, 최종 판단은 사람의 몫입니다.
4. **리포트 머리·꼬리 고지.** 첫머리에 "무료 공개 데이터 기반 학습용" 한 줄, 끝에 사용한 데이터 출처·기준일 목록을 답니다.
